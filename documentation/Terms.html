There are certain terms I have defined that corresponds to the working of this tool.

Macro Files
Client Sends
These are the actual requests from a client. A client would send a request or a notice to the server and the data sent would be considered a "client send".
Main Manifold/Core Program
This is the script which is compiled first. In this case its the WebTV.exe file. This program is what maintains control over the macro scripts and is the main client connection monitor.
Macro Extension
This is the script that the core program reads and compiles for execution. These scripts contain instructions that explain what to do with the client sends.
Turn Key File
This is the configuration extension for the server that controls the registration key. This file contains variables on how to start the program and identifies who is the owner of the program.
Mode 4 file Server

Client Program Flow
The program flow is the path which the client sends goes through before the client gets a response. In the file server Client Program Flow there are 3 different procedures which read, manipulate, reject, and write to a clients connection.
Primary Flow Control File
This is the file which does most of the work with determining manipulating and rejecting what the client sends are. Ex: DummyPerlTarget.wtv
Secondary Flow Control File
These are the files that assist the Primary Flow Control File in determing what to do with the client sends. Ex: ProtoFinder.pl
Tertiary Flow Control File
These are the final scripts that the Primary Flow Control and Secondary Flow Control Files have determined to reach. These are the files which send the response out to the client if there was not response before hand.
